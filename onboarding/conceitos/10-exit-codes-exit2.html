Exit codes e o significado do exit 2

O código de saída de um processo é um número entre 0 e 255 que o sistema operacional entrega a quem o chamou. No Nemesis, ele é o mecanismo de enforcement: 2 significa "bloqueado".

Em 30 segundos

Toda IDE de agente chama um hook antes de executar uma ação. Se o hook sai com 0, a ação prossegue; se sai com 2, a IDE entende "negado" e a ação não acontece. Não é um pedido, é um veto binário decidido por código.

Do zero: o que é um exit code

Quando um processo termina, ele devolve um inteiro ao processo-pai (no shell, fica em $?). Convenção POSIX: 0 = sucesso; qualquer valor diferente de zero = falha. O número específico é semântico, definido pela aplicação. As IDEs de agente padronizaram 2 como "bloqueie esta ação e mostre a razão ao modelo".

Por que importa

Um aviso em texto ("por favor não faça isso") é uma sugestão que o modelo pode ignorar. Um exit 2 é uma decisão tomada fora do modelo, em código determinístico, no momento exato anterior à ação. É a diferença entre persuasão e contenção. Ver probabilístico vs determinístico.

Como o Nemesis aplica

O hook pretool envolve toda a lógica num catch_unwind: saída normal de bloqueio é exit(2), e qualquer pânico inesperado também vira exit(2) (fail-closed).

📄 .nemesis/hooks/nemesis-pretool-check-unix.rs:164-194 — nemesis_block 
fn nemesis_block(reason: &str, instruction: Option<&str>) -> ! {
    eprintln!("{}", reason);
    if let Some(instr) = instruction { eprintln!("→ {}", instr); }
    // ledger só registra bloqueios de SEGURANÇA (prefixo "NEMESIS ")
    if reason.contains("NEMESIS ") {
        nemesis_defender::violations_log::append("pretool", reason);
    }
    std::process::exit(2);
}

O scanner pontual usa o mesmo número: --scan sai com 2 quando o veredito é Malicious.

Nuances e armadilhas

  • O tipo de retorno -> ! ("never") prova ao compilador que a função nunca retorna: ela sempre sai do processo. Isso é uma garantia verificada estaticamente.
  • Exit codes vão de 0 a 255; um exit(256) vira 0. Por isso convenções usam números pequenos.
  • O ledger só grava bloqueios de segurança; erros operacionais do hook não poluem a telemetria.

Auto-checagem

Por que exit 2 e não uma mensagem no prompt?

Porque a mensagem é probabilística (o modelo decide se obedece); o exit code é determinístico (o SO e a IDE obedecem). Contenção, não persuasão.

O que significa o tipo de retorno -> ! em nemesis_block?

É o tipo "never": a função nunca retorna ao chamador, sempre encerra o processo. Garantia em tempo de compilação.

Um pânico não tratado no hook libera a ação?

Não. O catch_unwind em main() converte qualquer pânico em exit(2) — falha fechada. Ver panic e fail-closed.
