AST, tree-sitter e o padrão Visitor

Uma AST representa o código como árvore estruturada, não como texto. Analisar a árvore (em vez de só casar regex) permite entender o que o código faz, não só como ele se parece.

Em 30 segundos

O tree-sitter transforma código JS/TS/Bash/Python em árvore. O Nemesis percorre essa árvore (travessia recursiva) e, em cada nó, chama os visitors, cada um especializado em um vetor de ataque. É análise estática semântica: entende chamadas, argumentos e estrutura.

Do zero

Uma AST (Abstract Syntax Tree) é a estrutura que um compilador monta: eval(decode(x)) vira um nó "chamada" cujo argumento é outra "chamada". O tree-sitter é uma biblioteca de parsers incrementais que produz essa árvore para dezenas de linguagens. O padrão Visitor separa "como percorrer a árvore" de "o que fazer em cada nó": você visita cada nó e despacha para handlers especializados.

Por que importa

Regex vê texto; AST vê semântica. "eval" num comentário não é perigoso; eval(...) como chamada de função é. A AST distingue os dois. É a diferença entre alarme falso e detecção real, e o que permite o modelo de severidade ter sinais confiáveis.

Como o Nemesis aplica

Para cada linguagem, um parser tree-sitter é criado e a raiz é percorrida recursivamente; em cada nó, todos os visitors relevantes são chamados:

📄 .nemesis/nemesis-defender/src/scanner/ast_scanner.rs:63-86 
fn traverse_javascript_node(node: &Node, source: &str, violations: &mut Vec, file_path: &str) {
    violations.extend(visitors::decode_exec::visit_js_node(node, source));
    violations.extend(visitors::dynamic_cmd::visit_js_node(node, source));
    violations.extend(visitors::url_in_exec::visit_js_node(node, source));
    violations.extend(visitors::unicode_steg::visit_js_node(node, source));
    visitors::prompt_injection::visit_js_node(node, source); /* ...e outros... */
    let mut cursor = node.walk();
    for child in node.children(&mut cursor) {
        traverse_javascript_node(&child, source, violations, file_path);
    }
}

Cada visitor é um módulo em src/visitors/ que recebe um nó e devolve Vec<DefenderViolation>. São 15 ao todo, cobrindo JS/TS, Bash e Python conforme a linguagem.

Nuances e armadilhas

  • Tecnicamente o tree-sitter produz uma CST (árvore concreta) que o scanner usa como AST. O código chama de AST por convenção.
  • Só JS/TS, Bash e Python têm AST. Outros tipos (JSON, TOML, Markdown) passam por bytes/regex/manifesto, sem parser.
  • O taint tracker não é um visitor de nó: ele roda uma passada full-content após a travessia (análise de fluxo entre linhas). Ver taint tracking.

Auto-checagem

Por que AST detecta melhor que regex pura?

Porque enxerga a estrutura: distingue eval(...) chamada real de "eval" em comentário/string, reduzindo falsos positivos.

O que o padrão Visitor separa?

A travessia da árvore (genérica) da ação por nó (cada visitor especializado num vetor).

Quais linguagens têm análise AST no Nemesis?

JavaScript/TypeScript, Bash e Python (parsers tree-sitter correspondentes).
