Self-clean, time-gated e Unicode steganography

Três técnicas de furtividade: apagar os próprios rastros, esperar o momento certo para atacar, e esconder código com caracteres invisíveis ou enganosos.

Em 30 segundos

Self-clean: o malware se deleta após executar, para evitar perícia. Time-gated: o payload só dispara depois de uma data ou delay, passando despercebido em testes. Unicode steg: caracteres BiDi/PUA/homoglyph fazem o código exibido na tela diferir do que o compilador executa (Trojan Source, CVE-2021-42574).

Do zero

Self-cleaning usa fs.unlink(__filename) ou rm $0 para sumir. Time-gated usa setTimeout/comparações de data para adiar a carga. Esteganografia Unicode abusa de caracteres especiais: controles BiDi reordenam a exibição; homóglifos (letras de outro alfabeto que parecem ASCII) enganam o revisor; caracteres invisíveis escondem dados.

Por que importa

Essas técnicas atacam a confiança visual e a janela de detecção. Um humano revisa o código e vê algo benigno; o que executa é outra coisa, ou só vira hostil depois. A defesa precisa olhar os bytes, não a renderização, e marcar gatilhos temporais.

Como o Nemesis aplica

O byte scanner separa controles BiDi perigosos (confirmatório unicode_bidi) de marcas fracas i18n/emoji (corroborante unicode_zero_width):

📄 .nemesis/nemesis-defender/src/scanner/byte_scanner.rs:20-47, 62-98 
const BIDI_DANGEROUS: &[(u32, &str)] = &[
    (0x202E, "Right-to-Left Override"), (0x2066, "Left-to-Right Isolate"), /* ... */
];   // sem uso legítimo em código → confirmatório (Trojan Source / Glassworm)
const BIDI_WEAK: &[(u32, &str)] = &[
    (0x200E, "Left-to-Right Mark"), (0xFE0F, "Variation Selector-16"), /* emoji/i18n */
];   // ambíguo → corroborante (exige 2º sinal)

self_clean (corroborante) detecta unlink(__filename) e reescrita de package.json; time_gated (corroborante) detecta setTimeout/data com ramo malicioso.

📄 .nemesis/nemesis-defender/src/visitors/self_clean.rs · time_gated.rs · unicode_steg.rs

Nuances e armadilhas

  • A distinção dangerous/weak é calibração fina: emoji legítimo (❤️ = U+2764 U+FE0F) não pode deletar arquivo. Por isso marcas fracas são corroborantes.
  • Glassworm (2025) abusa de variation selectors, mas sempre junto de outro payload, então o modelo de corroboração o pega via 2 sinais.
  • Homóglifos em identificadores ASCII (spoof de nome de função) são confirmatórios (unicode_homoglyph_ident, CVE-2021-42694); homóglifo isolado é corroborante.

Auto-checagem

O que é Trojan Source?

Uso de controles BiDi para fazer o código exibido diferir do compilado (CVE-2021-42574), enganando o revisor humano.

Por que separar BiDi "dangerous" de "weak"?

Porque marcas direcionais e variation selectors aparecem em i18n/emoji legítimos; tratá-los como confirmatórios geraria falsos positivos.

Por que self-clean e time-gated são corroborantes?

Porque isoladamente podem ter usos legítimos (limpeza de temp, agendamento); exigem um segundo sinal para confirmar malícia.
